Secciones y lineas generales de un documento HTML5

La especificacion HTML5 trae muchos nuevos elemntos a los desarrolladores web, permitiendoles describir la estructura de un documento web con la semantica estandarizada. Este documento describe estos elemntos y como utilizarlos para definir el perfil de cualquier documento.

Problemas resueltos por HTML5

La definicion de la estructura de unu documento HTML4. y su algoritmo de perfilado es muy tosco y genera numerosos problemas:

  1. HTML5 quita la necesidad de elementos <div> para definir secciones semanticas sin definir valores especificos para los atributos class, introduciendo un nuevo elemento, <section>, el elemento de seccion HTML.
  2. Mezclar varios elemntos es dificil: la inclusión de un sub-documento en docuemnto principal. Esto se resuelve en HTML5 con los elemntos de seccionado (<article>,<section>, <nav> y <aside>) son siempre subsecciones de su seccion ancestra mas cercana.
  3. HTML5 introduce el elemnto
    que oculta todos los elemtos de cabecera excepto el primero de más alto rango (por ejemplo, <hgroup><h1> Justine </h1> <h2> Les Malheurs de la Vertu < </group> crea el perfil 1 Justine).
